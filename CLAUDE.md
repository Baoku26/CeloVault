## Project identity

**CVault** (formerly CeloVault) is an autonomous multi-currency stablecoin wallet agent on Celo.
It tracks balances, auto-swaps at best rates, and pays in local currency.
Registered as a first-class ERC-8004 onchain agent.

- **Hackathon:** Celo Onchain Agents Hackathon

## Fee abstraction rule

**All agent transactions must use `feeCurrency`.**
The agent wallet should never hold CELO. Gas is paid in USDC.

```typescript
// Always use the ADAPTER address, not the token address for USDC/USDT
const hash = await walletClient.writeContract({
  ...contractConfig,
  feeCurrency: USDC_ADAPTER,  // ← required on every agent tx
});
```

For 18-decimal tokens (USDm, NGNm, etc.) use the token address directly as feeCurrency.
For 6-decimal tokens (USDC, USDT) always use the adapter address.

## UI design rules

> CeloVault is a **premium dark-mode dashboard** with a fintech edge.
> It is NOT a generic crypto dApp. It must look like something you'd pay for.

### Non-negotiables

1. **Dark mode only.** Background is near-black (`#0A0A0A` or `zinc-950`). Never white.
2. **No purple gradients.** No generic "web3" aesthetics. No glowing orbs.
3. **Typography is editorial.** Use `Geist` or `DM Mono` for numbers, a sharp sans for UI labels.
4. **Celo green (`#35D07F`) is the accent.** Secondary accent: `#FBCC5C` (Celo gold). Used sparingly — not everywhere.
5. **No stock card layouts.** Every component should feel purpose-built for financial data.
6. **Numbers breathe.** Financial figures get generous sizing, tabular-nums, and proper alignment.
7. **Micro-interactions matter.** Framer Motion entrance animations on balance cards, swap confirmations, and agent status changes.

### shadcn/ui usage

- Components live in `components/ui/` — you OWN the code, modify freely.
- Never import from `@shadcn/ui` NPM package — copy-paste the component code.
- Extend components to add `data-slot` attributes, custom variants, and motion wrappers.
- Prefer `cva` for variant management on all custom components.

### Impeccable workflow

When building UI, follow this sequence:
1. `/impeccable shape` — design brief before building any new screen
2. `/impeccable craft` — build it
3. `/impeccable critique` — review it
4. `/impeccable polish` — final pass before shipping

For existing screens:
- `/impeccable bolder` — if something feels too timid
- `/impeccable typeset` — if typography feels generic
- `/impeccable delight` — to add personality moments
- `/impeccable animate` — purposeful motion, not decoration

The `DESIGN.md` file is the visual source of truth. Run `/impeccable document` to regenerate it after major design changes.

### Component structure rules

Every component file follows this pattern:

```tsx
// components/wallet/BalanceCard.tsx

import { cn } from "@/lib/utils"
import { cva, type VariantProps } from "class-variance-authority"
import { motion } from "framer-motion"

// 1. Variants defined at top with cva
const balanceCardVariants = cva("...", {
  variants: { ... },
  defaultVariants: { ... },
})

// 2. Props interface extends VariantProps
interface BalanceCardProps extends VariantProps<typeof balanceCardVariants> {
  token: TokenMetadata
  balance: string
  usdValue: string
  className?: string
}

// 3. Component with motion wrapper
export function BalanceCard({ token, balance, usdValue, className, ...props }: BalanceCardProps) {
  return (
    <motion.div
      initial={{ opacity: 0, y: 8 }}
      animate={{ opacity: 1, y: 0 }}
      className={cn(balanceCardVariants(props), className)}
    >
      ...
    </motion.div>
  )
}
```

## Agent loop rules

The standalone agent (`agent/index.ts`) must:

1. Run on a configurable interval (default: 60 seconds)
2. Read balances for all configured tokens
3. Query best rate from Mento, fall back to Uniswap V3
4. Compare rate against user-configured threshold
5. Execute swap if threshold is met AND balance >= minimum
6. Pay gas in USDC via feeCurrency adapter
7. Submit ERC-8004 reputation feedback after every successful swap
8. Log all activity to stdout in structured JSON format
9. Expose `/health` HTTP endpoint (Railway health check)
10. Be fully stateless — restartable at any time without data loss

## Coding standards

- **TypeScript strict mode** everywhere. No `any`. No `as unknown as`.
- All blockchain addresses are typed as `` `0x${string}` ``.
- All token amounts are `bigint` (never `number` for onchain values).
- All async functions have explicit return types.
- Errors are caught and surfaced to UI via `sonner` toast notifications.
- No inline styles. Tailwind only. Custom values go into `tailwind.config.ts` theme extension.
- File names: `PascalCase` for components, `camelCase` for lib/hooks/utils.
- Every new `lib/` module exports a named barrel from `lib/index.ts`.
